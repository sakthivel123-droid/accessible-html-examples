WCAG 2.2 – 1.1.1 Non-text Content

This page demonstrates good and bad examples of providing text alternatives for non-text content such as images, icons, charts, and controls.

❌ Bad Example 1: Image without alt text

Issue: No alt attribute provided. Users with screen readers won’t know what this image represents.

✅ Good Example 1: Image with proper alt text

Fix: Added a meaningful alt text describing the image’s content and purpose.

❌ Bad Example 2: Decorative image with alt text

Issue: Decorative images should have alt="" or be in CSS. This one distracts screen reader users unnecessarily.

✅ Good Example 2: Decorative image handled correctly

Fix: Used alt="" so screen readers skip it. Decorative images should not add noise.

❌ Bad Example 3: Image used as button without text alternative

Issue: Screen reader users won’t know this is a “Search” button.

✅ Good Example 3: Image button with accessible name

Fix: Added alt text or aria-label describing the control’s purpose (“Search”).

❌ Bad Example 4: CAPTCHA without accessible alternative

Issue: CAPTCHA requires visual identification and provides no text/audio alternative.

✅ Good Example 4: CAPTCHA with accessible alternative

Can’t see the image? Try the audio CAPTCHA.

Fix: Provided a text instruction and an audio alternative for users who cannot see the image.

❌ Bad Example 5: Complex chart with no text description

Issue: The alt text is too generic and doesn’t convey the chart’s data or purpose.

✅ Good Example 5: Complex chart with detailed text alternative

Monthly Sales Data (2025):

  • January: $12,000
  • February: $15,000
  • March: $9,500

Fix: Provided a short alt text plus a full text equivalent describing the same data for users who can’t see the chart.
