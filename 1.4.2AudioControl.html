WCAG 2.2 – 1.4.2 Audio Control

Guideline: If audio plays automatically for more than 3 seconds, provide a way to pause, stop, or mute it.

Bad Example: Auto-playing background music with no controls

This example fails WCAG 1.4.2 because audio starts automatically and has no pause/stop/mute option. It interferes with screen readers and user control.

Good Example: Audio with user control options

This example meets WCAG 1.4.2 because users can start or stop audio manually. No sound plays automatically.

Good Example: Auto-playing audio with pause/mute controls provided

This example also passes because autoplay audio includes a visible pause and mute button.

Explanation of Scenarios

  • Scenario 1 – Autoplay without control: Fails because user cannot stop audio. Screen readers are disrupted.
  • Scenario 2 – Manual play (controls attribute): Passes because no sound starts automatically.
  • Scenario 3 – Autoplay with control: Passes because user can pause, stop, or mute the sound anytime.
  • Scenario 4 – Audio shorter than 3 seconds: Passes even if it plays automatically (e.g., notification sound).

Note: This criterion applies to any sound longer than 3 seconds that starts automatically. Background music, voice announcements, or sound effects should all provide a way to control audio.
