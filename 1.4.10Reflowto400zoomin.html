WCAG 2.2 - 1.4.10 Reflow (400% Zoom)

❌ Bad Example: Fixed Layout (Fails Reflow)

This layout uses fixed widths and does not adjust when zoomed to 400% — users must scroll horizontally to read all content.

Box 1

This box has a fixed width. When zoomed, text overflows and requires horizontal scrolling.

Box 2

Users on small screens or 400% zoom cannot read content easily because of rigid structure.

Box 3

Breaks accessibility because it ignores user zoom and viewport size.

✅ Good Example: Responsive & Reflow-Friendly

This layout uses flexible boxes and percentages, allowing text to reflow vertically when zoomed up to 400%.

Box 1

Resize or zoom this page up to 400%. This box will move below others automatically instead of forcing a horizontal scroll.

Box 2

Flexible design ensures that all text remains readable and fits within the viewport width.

Box 3

Uses CSS flex-wrap and relative widths to support reflow without losing context.

Some content, like large data tables or maps, may require two-dimensional scrolling — this is allowed by WCAG.
