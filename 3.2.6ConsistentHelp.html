WCAG 2.2 - 3.2.6 Consistent Help

❌ Bad Example (Fails WCAG 3.2.6)

This is the Home Page. On this page, help is provided inconsistently.

Need Help?

Issues:

  • Help link position changes across pages (top-left here, bottom-right elsewhere).
  • Label text changes (“Need Help?” vs “Contact Us”).
  • Help type changes (sometimes chat, sometimes email link).
  • Screen reader users cannot predict where to find help.

If a page within a set of web pages provides help
(like contact support, chat, FAQs, or tooltips),
that help mechanism must appear in the same relative location
and be available consistently across all pages where it is provided.

✅ Good Example (Passes WCAG 3.2.6)

This is the Home Page. Here, help is consistent in placement, labeling, and type across all pages.

Need support?

  • Email Support
  • Call Us
  • Live Chat

Accessible Features:

  • Help button appears in the same position (bottom-right) on every page.
  • Same label “Help” used across all pages (not “Need Help?” or “Contact Us”).
  • Same interaction type (tooltip dialog with email, call, chat options).
  • Keyboard accessible (Tab and Enter activate it).
  • Screen readers announce it as a dialog (via role="dialog").
