WCAG 2.2 - 3.1.1 Language of Page & 3.1.2 Language of Parts

🚫 Bad Example - Language of Page (3.1.1)

Issue: The page doesn't specify the main language using lang attribute. Screen readers won’t know how to pronounce the text correctly.

Issue: Invalid language code (‘engl’) not recognized by assistive technologies.

🚫 Bad Example - Language of Parts (3.1.2)

Here’s a mixed-language sentence: Bonjour! How are you? Gracias!
Issue: The French (“Bonjour”) and Spanish (“Gracias”) parts are not marked with their respective language codes. Screen readers will pronounce them incorrectly.

✅ Good Example - Language of Page (3.1.1)

This page correctly uses <html lang="en"> so screen readers know to read in English.

Explanation: This embedded page sets lang='fr' correctly to indicate it’s French content.

✅ Good Example - Language of Parts (3.1.2)

Here’s a properly marked mixed-language sentence:

Bonjour! How are you? Gracias!

Explanation: The words “Bonjour” and “Gracias” are wrapped in elements with the correct lang attributes (fr for French, es for Spanish).
Screen readers will switch pronunciation accordingly.

📘 Summary of All Scenarios

  • 3.1.1 Language of Page: Must always have a valid lang attribute in the <html> tag.
  • 3.1.1 Failure Example: Missing or invalid language code (like “engl”).
  • 3.1.2 Language of Parts: Text in other languages must be wrapped in an element with that language’s lang attribute.
  • 3.1.2 Failure Example: Foreign words not marked with lang attributes cause wrong pronunciation.
