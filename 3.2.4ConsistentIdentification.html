WCAG 2.2 – 3.2.4 Consistent Identification

🚫 Bad Example – Inconsistent Labels for Same Function

Each of the following inputs does the same thing – search – but is labeled differently.


Issue: The same function (search) is labeled inconsistently: “Find Articles”, “Look Up Content”, “Go”, “Submit”. Users with assistive tech can’t predict that these do the same task.

🚫 Bad Example – Different Icons for Same Meaning

Here, the same "print" function uses two completely different icons.

Issue: Users may not realize both icons perform the same “print” action. Visual or cognitive users expect consistent symbols for repeated tasks.

✅ Good Example – Consistent Labels and Icons

Here, the same “search” function appears twice on a page – same label, same icon, consistent behavior.


Explanation: Both instances use the same label “Search”, same icon (🔍), and same button name “Search”. This consistency helps everyone recognize the same action anywhere on the site.

✅ Good Example – Consistent ARIA Labels for Same Components

Explanation: Even though the buttons appear in different areas, they use the same accessible name (“Download File”) ensuring screen readers identify both the same way.

📘 Summary of All Scenarios

  • 3.2.4 Consistent Identification: Same functions = same name, icon, label, and role.
  • Bad Scenarios:
    • Different labels (“Find”, “Search”, “Lookup”) for the same function.
    • Different icons for same task (🖨️ vs 📄 both for print).
    • Different ARIA labels for identical buttons.
  • Good Scenarios:
    • Consistent visual text and ARIA labels.
    • Identical iconography for same purpose.
    • Predictable and uniform interaction behavior across pages.
