WCAG 2.2 - 1.4.5 Images of Text

❌ Bad Example: Using an Image of Text

This banner below uses an image to display text, instead of real HTML text:

Problems:

  • Text inside the image cannot be resized properly without losing quality.
  • Screen readers cannot read or interpret font size, style, or color from the image.
  • Users cannot change color contrast or background based on their needs.

✅ Good Example: Using Real Text with CSS

This banner uses actual text styled with CSS, which maintains accessibility and flexibility:

Welcome to Our Store

Advantages:

  • Text can be resized up to 200% without losing clarity.
  • Screen readers can read it as normal text.
  • Users can change color schemes and high contrast modes easily.

✅ Exception Example: Logo or Branding

Images of text are allowed when the presentation of the text is essential (e.g., logos or stylized branding).

This is acceptable because logos are considered essential images of text and their appearance is part of the brand identity.

✅ Exception Example: Text within Complex Images

Images that include text as part of essential content (like graphs, charts, or screenshots) are permitted.

Here, the text 'Sales 2025' is part of the chart and essential to understanding the data — this is acceptable.
